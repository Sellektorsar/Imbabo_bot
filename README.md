# Telegram Bot @imbabo_bot_v2

Telegram-бот для продажи солнцезащитных и имиджевых очков с использованием маркетинговой модели AIDA.

## 🚀 Возможности

### Пользовательские функции
- 🕶️ **Каталог товаров** - просмотр категорий и товаров с пагинацией
- 🎯 **Персональный подбор** - интерактивный подбор очков по критериям
- 🛒 **Корзина и заказы** - добавление товаров, оформление заказов
- 🎫 **Промокоды** - система скидок для первой покупки
- 💬 **Отзывы** - сбор текстовых и голосовых отзывов
- ❓ **FAQ** - часто задаваемые вопросы
- 📢 **Проверка подписки** - ограничение доступа без подписки на канал

### Административные функции
- 📊 **Статистика** - аналитика пользователей, продаж, конверсии AIDA
- 📦 **Управление каталогом** - добавление/редактирование товаров и категорий
- 📋 **Управление заказами** - просмотр и изменение статусов заказов
- 🎫 **Управление промокодами** - создание и настройка промокодов
- 📧 **Рассылки** - создание и отправка маркетинговых сообщений
- ❓ **Управление FAQ** - редактирование вопросов и ответов
- 👥 **Управление пользователями** - просмотр данных пользователей

### Автоматические функции
- 📅 **Ежедневные рассылки** - автоматическая отправка в 10:00
- 🛒 **Напоминания о корзине** - уведомления через 2 часа после добавления товара
- 📢 **Автопостинг в канал** - публикация контента каждые 4 часа
- 🧹 **Очистка данных** - еженедельная очистка старых записей

## 🛠 Технологический стек

- **Python 3.8+** - основной язык программирования
- **aiogram 3.4.1** - фреймворк для Telegram-ботов

## 📦 Установка и настройка

### 1. Клонирование репозитория
```bash
git clone <repository-url>
cd imbabo_bot_v2
```

### 2. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 3. Настройка переменных окружения
Создайте файл `.env` в корне проекта:

```env
# Telegram Bot
BOT_TOKEN=your_bot_token_here
ADMIN_IDS=123456789,987654321

# Database
DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/imbabo_bot

# Channel
CHANNEL_USERNAME=your_channel_username

# Logging
LOG_LEVEL=INFO

# Optional
SUPPORT_USERNAME=your_support_username
```

### 4. Настройка базы данных

#### Установка PostgreSQL
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install postgresql postgresql-contrib

# Создание базы данных
sudo -u postgres createdb imbabo_bot
sudo -u postgres createuser imbabo_user
sudo -u postgres psql -c "ALTER USER imbabo_user PASSWORD 'your_password';"
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE imbabo_bot TO imbabo_user;"
```

#### Инициализация миграций (опционально)
```bash
# Инициализация Alembic
alembic init alembic

# Создание первой миграции
alembic revision --autogenerate -m "Initial migration"

# Применение миграций
alembic upgrade head
```

### 5. Запуск бота
```bash
python main.py
```

## 🔧 Конфигурация

### Основные настройки в config.py:
- `BOT_TOKEN` - токен Telegram-бота
- `ADMIN_IDS` - ID администраторов (через запятую)
- `DATABASE_URL` - строка подключения к PostgreSQL
- `CHANNEL_USERNAME` - username канала для проверки подписки
- `LOG_LEVEL` - уровень логирования (DEBUG, INFO, WARNING, ERROR)

### Настройка планировщика:
- Ежедневные рассылки: 10:00 UTC
- Напоминания о корзине: каждые 2 часа
- Автопостинг в канал: каждые 4 часа
- Очистка данных: еженедельно в воскресенье

## 📊 Модель AIDA

Бот реализует маркетинговую воронку AIDA:

1. **Attention (Внимание)** - эмоциональное приветствие, автопостинг в канал
2. **Interest (Интерес)** - каталог товаров, персональный подбор
3. **Desire (Желание)** - метафорические описания, промокоды, отзывы
4. **Action (Действие)** - простое оформление заказа, напоминания о корзине

## 🔐 Безопасность

- Административные функции доступны только пользователям из `ADMIN_IDS`
- Проверка подписки на канал для доступа к основным функциям
- Логирование всех действий пользователей и администраторов
- Обработка ошибок с уведомлением администраторов

## 📝 Логирование

Система логирования записывает:
- Действия пользователей
- Административные операции
- События заказов
- События рассылок
- Ошибки и исключения
- Метрики производительности

Логи сохраняются в файл `logs/bot.log` и выводятся в консоль.
